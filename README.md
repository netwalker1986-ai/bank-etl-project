# Итоговый ETL-проект

## Описание
Проект реализует ETL-процесс загрузки банковских данных из файлов в PostgreSQL с последующим формированием витрины мошеннических операций.

Обрабатываются три типа файлов:
- transactions_DDMMYYYY.txt
- passport_blacklist_DDMMYYYY.xlsx
- terminals_DDMMYYYY.xlsx

После загрузки данные попадают в STG-слой, затем переносятся в DWH-слой, после чего формируется итоговая витрина `rep_fraud`.

## Структура проекта
- `main.py` — основной Python-скрипт запуска ETL-процесса
- `sql_scripts/` — SQL-скрипты для создания таблиц и обработки данных
- `archive/` — папка для архивирования обработанных файлов
- `transactions_*.txt` — входные файлы транзакций
- `passport_blacklist_*.xlsx` — входные файлы blacklist
- `terminals_*.xlsx` — входные файлы терминалов

## Используемые технологии
- Python
- PostgreSQL
- psycopg
- pandas
- openpyxl
- python-dotenv

## Логика работы
1. Python-скрипт считывает входные файлы из корня проекта.
2. Данные загружаются в STG-таблицы.
3. Данные переносятся в DWH-таблицы.
4. После загрузки формируется витрина мошеннических операций `rep_fraud`.
5. Обработанные файлы переименовываются в `.backup` и перемещаются в папку `archive`.

## Формируемая витрина
Итоговая таблица:
- `rep_fraud`

Поля витрины:
- `event_dt`
- `passport`
- `fio`
- `phone`
- `event_type`
- `report_dt`

## Реализованные правила поиска мошеннических операций
1. Операция при просроченном или заблокированном паспорте
2. Операция при недействующем договоре
3. Операции в разных городах в течение одного часа
4. Попытка подбора суммы

## SQL-скрипты
